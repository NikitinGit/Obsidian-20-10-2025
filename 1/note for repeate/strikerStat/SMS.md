# SMS в проекте StrikerStat

Всё, что связано с отправкой SMS: шлюз, OTP, уведомления о вызовах и (главное) модерация шаблонов у провайдера. Провайдер — **smsint.ru** (личный кабинет `https://lcab.smsint.ru/`).

## Где в проекте SMS

| Место | Назначение |
|---|---|
| `service/auth/AuthSmsService` | Клиент SMS-шлюза smsint.ru (единственная точка реальной отправки) |
| `service/auth/PhoneOtpSendService` | OTP-коды по SMS (вход/регистрация) |
| `service/auth/AuthFlashCallService` | Альтернатива OTP — флеш-звонок (zvonok.com), не SMS, но тот же сценарий «код» |
| `service/challenge/ChallengeNotificationService` | Ставит SMS-уведомление о вызове в очередь |
| `service/challenge/ChallengeNotificationWorker` | `@Scheduled` воркер: реально шлёт SMS/email из очереди |
| `controller/admin/AdminPanelTechnicalTestsController` | Админка: тест отправки OTP, статус/баланс шлюза |

**Важно:** и OTP, и уведомления о вызовах шлют через **один и тот же** `AuthSmsService.send(text, phone)`. Отличается только текст. Это ключ к пониманию проблемы с шаблонами (см. ниже).

## AuthSmsService — шлюз smsint.ru

```java
SmsResult send(String text, String phone)   // POST json/v1.0/sms/send/text
Double   getBalance()                        // POST json/v1.0/cabinet/account/getBalance
boolean  isConfigured()                      // token задан?
```

- Тело запроса: `{"messages":[{"recipient": phone, "text": text}]}`, заголовок `X-Token: <token>`. **Никакого id шаблона/отправителя в запросе нет** — шлюз сам матчит текст по зарегистрированным шаблонам.
- Конфиг:
  - `app.auth.sms.base-url` (дефолт `https://lcab.smsint.ru/`)
  - `app.auth.sms.token` / env `SMS_TOKEN`
- Маппинг ответа в `SmsResult(status, gatewayCode, gatewayDescription)`:
  - success → `OK`
  - код **517** → `CANNOT_BE_DELIVERED` (постоянная ошибка)
  - всё остальное (в т.ч. **520**) → `UNKNOWN`

### Зависимости (pom.xml)

**Отдельной SMS-библиотеки/SDK нет** — REST-API smsint.ru дёргается напрямую. Используются:
- `spring-boot-starter-web` — даёт `RestTemplate` (+ `HttpHeaders`/`HttpEntity`/`MediaType`) для HTTP-вызова шлюза.
- `jackson-databind` (транзитивно из starter-web) — `ObjectMapper`/`JsonNode` для сборки тела запроса и разбора ответа.
- `lombok` — `@RequiredArgsConstructor`/`@Slf4j` (не SMS-специфично).

То есть «зависимость для SMS» = обычный Spring REST-клиент, не сторонний пакет.

## OTP по SMS (PhoneOtpSendService)

**OTP = One-Time Password** — одноразовый пароль (код подтверждения, действует один раз/ограниченное время; используется для входа и регистрации по телефону).

- Текст: `Код подтверждения на сайте Strikerstat: <код>` — этот шаблон **зарегистрирован и одобрен** у провайдера, поэтому OTP работает.
- Флаг **`app.auth.mock_sms`** (дефолт `false`): если `true` — реальный шлюз не дёргается, код пишется в лог (`SMS mock result: ... Код: X`) и считается доставленным. **Так логинятся локально** без настроенного шлюза.
  - ⚠️ Этот mock — **только в `PhoneOtpSendService`**, у challenge-воркера его НЕТ (он всегда идёт в реальный шлюз).

## SMS-уведомления о вызовах (challenge)

Поток: действие ПА → `ChallengeNotificationService.notify(...)` → запись в очередь → воркер шлёт.

- **Очередь** — таблица `event_challenge_notifications` (`is_sent=false`). Это outbox: пишется атомарно с доменным изменением, отправка — позже, вне транзакции пользовательского запроса (внешний I/O в транзакции БД нельзя).
- **SMS ставится в очередь только на `PENDING`** (новый вызов), получатель = тот, кому бросили вызов (`challengedFighter`). На принятие/отклонение/отзыв SMS НЕ шлём (текст «Вам бросили вызов» туда не подходит) — только in-app/email.
- **Воркер `ChallengeNotificationWorker`** (`@Scheduled fixedDelay`, конфиг `jobs.challenge-notifications.{enabled,fixed-delay,initial-delay}`). На каждую запись:
  - нет `User` у бойца → `log.error` (аномалия данных) + снять из очереди;
  - канал выключен у получателя / нет телефона → снять без доставки;
  - **«спал»**: последняя `battles.modified_at` > 365 дней → снять без доставки;
  - **лимиты раздельно по каналам**: 1/сутки, 2/неделю, 3/месяц (скользящие окна) → отложить;
  - шлюз не настроен (`isConfigured`) → отложить;
  - иначе отправка. **Best-effort: после реальной попытки — БЕЗ повторов** (успех → `is_sent=true,sent_at=now`; неудача → снять из очереди, `sent_at=null`). Сделано специально, чтобы постоянные ошибки (битый адрес, незарегистрированный шаблон) не ретраились каждый тик и не засоряли лог. Надёжный канал — in-app (доставлен сразу).
- **Текст** — i18n `challenge.notification.sms.message`, только в `messages.properties` (ru). En НЕ заполняем: SMS формируется в воркере без запроса (`LocaleContextHolder` = дефолт JVM), а при отсутствии en-ключа всё резолвится в базовый ru через fallback (`I18nConfig`: `fallbackToSystemLocale=false`, базовый бандл `messages/messages`). Заполнить en = риск английского SMS.

### Финальный текст (решение команды)

```
Вам бросили вызов на соревнование {0}. Подробнее на Strikerstat.
```
- `{0}` = **дата соревнования** в формате `dd.MM.yyyy` (воркер: `eventDateOf`, форматтер `SMS_DATE_FORMAT`).
- Дату взяли вместо названия, потому что: (1) укладывается в **70 символов** одного SMS-сегмента (кириллица = UCS-2, 70 симв./сегмент); (2) предсказуемый формат (цифры) надёжно ложится на переменную шаблона, в отличие от свободного названия.

## Шаблоны smsint.ru — модерация (главная грабля)

Российские агрегаторы шлют **только по заранее зарегистрированным и одобренным операторами шаблонам**. Если текст не совпал с шаблоном:

```
SMS-шлюз вернул ошибку: code=520, descr=Шаблон для рассылки не найден
```

> [!warning] 520 ≠ баг кода
> Код шлёт SMS одинаково для OTP и уведомлений. OTP работает, потому что его шаблон зарегистрирован; новый текст уведомления — нет → 520. Лечится **только регистрацией шаблона в ЛК провайдера**, не кодом.

### Где регистрировать
ЛК → раздел **«Шаблоны сообщений»** (`https://lcab.smsint.ru/cabinet/pager/preTemplates`) → **«Добавить шаблон»**. Доступ к ЛК — у админа/devops. У одобренных шаблонов статус «Одобрен».

### Что должно совпасть
1. **Имя отправителя (Sender ID)** — то же, под которым работает OTP (шаблон привязан к одобренному отправителю).
2. **Статическая часть текста = то, что шлёт бэкенд, символ-в-символ.** Любой лишний/недостающий пробел → 520.
   - Реальный баг: в шаблоне было `соревнование [w] . Для...` (пробел перед точкой), а бэкенд слал `...{0}. Для...` (без пробела) → 520. Лечится подгонкой одного к другому.
3. **Переменная** на месте подставляемого значения.

### Синтаксис переменных (по примеру их шаблона)
- `[d]` = **число/цифры** (даты, телефон): `с [d] по [d]`, `тел. [d]`
- `[w]` = **слово/текст**: `скидка [w] на все`
- У переменной есть параметры, напр. ограничение по числу слов:
  ```
  [w(variable-params){"count":{"=":[{"min":"1","max":"3"}]}}(/variable-params)]
  ```
  (`[w]` от 1 до 3 слов)

> [!danger] Свободный текст (название ОР) плохо ложится на шаблон
> `[w]{1,3}` ломается, если в значении: **>3 слов**, **цифры** (год «Кубок 2026»), **знаки** (кавычки, тире, скобки). Реальные названия почти всегда содержат что-то из этого → 520. Поэтому ушли от названия к **дате** (`[d]`-подобная переменная, фиксированный формат).

> [!tip] 70 символов
> Кириллица в SMS кодируется UCS-2 → **70 символов на один сегмент**. Длиннее — разбивается на несколько сегментов (дороже, и сложнее матчить шаблон). Короткий текст с датой — осознанный выбор под этот лимит.

## Админка — тесты SMS

`/api/admin-panel/technical-tests` (контроллер `AdminPanelTechnicalTestsController`). Доступ — пользователь с **`users.acstatus = 'orgAdmin'`** (security на `/api/admin-panel/**` = просто `authenticated()`, реальная проверка `requireAdminAccess` в сервисах). Фронт: `pages/admin-panel/index.vue` (`http://localhost:3000/admin-panel`).

- `GET /status` → `smsConfigured`, `balance`, отправитель, `mailConfigured`, `publicBaseUrl`…
- `POST /otp` → тест отправки OTP (SMS/флеш-звонок), возвращает код/статус.

## Локально: как протестировать вход по SMS

1. `app.auth.mock_sms=true` в `application-local.properties` (профиль `local`).
2. Войти по телефону через UI → код взять из **лога бэкенда** (`SMS mock result: ... Код: X`).
3. Чтобы попасть в админку — в БД `UPDATE users SET acstatus='orgAdmin' WHERE login='<твой логин/телефон>'`.

## Чеклист: «SMS не приходит»

1. `code=520` → **текст не совпал с одобренным шаблоном**: сверить статическую часть символ-в-символ (пробелы!), проверить Sender ID, проверить что переменная покрывает значение (формат/длина/символы).
2. `code=517` (`CANNOT_BE_DELIVERED`) → постоянная ошибка номера, доставки не будет.
3. Нет ошибки шлюза вовсе, но и SMS нет → проверить `isConfigured` (токен), баланс (`/status`), не выключен ли воркер (`jobs.challenge-notifications.enabled`), не сработал ли лимит/«спал»/опт-аут.
4. Локально challenge-SMS не идут, а OTP «работает» → у OTP есть `mock_sms`, у воркера нет; воркер бьётся о реальный шлюз.
