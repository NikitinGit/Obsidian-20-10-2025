# Поразрядные операции в Kotlin

В Kotlin **нет операторов** `&`, `|`, `^`, `~`, `<<`, `>>`, `>>>` — вместо них инфиксные функции.

| Java | Kotlin | Что делает |
|---|---|---|
| `a & b` | `a and b` | побитовое И |
| `a \| b` | `a or b` | побитовое ИЛИ |
| `a ^ b` | `a xor b` | исключающее ИЛИ |
| `~a` | `a.inv()` | инверсия битов |
| `a << n` | `a shl n` | сдвиг влево |
| `a >> n` | `a shr n` | сдвиг вправо (знаковый) |
| `a >>> n` | `a ushr n` | сдвиг вправо (беззнаковый) |

Доступны для `Int` и `Long`. Для `Byte`/`Short` нужно `.toInt()`.

---

>[!question]- and — побитовое И
>Если оба бита 1 — результат 1, иначе 0.
>```
>0101
>1101 and
> = 0101
>```
>```kotlin
>val r = 0b0101 and 0b1101    // 5
>```

>[!question]- or — побитовое ИЛИ
>Если хотя бы один бит 1 — результат 1.
>```
>0101
>1101 or
> = 1101
>```
>```kotlin
>val r = 0b0101 or 0b1101    // 13
>```

>[!question]- xor — исключающее ИЛИ
>Если биты разные — 1, иначе 0.
>```
>0101
>1101 xor
> = 1000
>```
>```kotlin
>val r = 0b0101 xor 0b1101    // 8
>```

>[!question]- inv — инверсия
>Каждый бит инвертируется (унарная операция).
>```
>0101
> inv
> = ...11111010 (все остальные биты тоже)
>```
>```kotlin
>val r = 0b0101.inv()    // -6 (для Int)
>```

>[!question]- shr — сдвиг вправо (знаковый)
>Все биты вправо на n. Деление на 2^n. Слева — знаковый бит копируется.
>```
>1010 shr 1 = 0101
>```
>Остаток (-1) при нечётном отбрасывается.

>[!question]- shl — сдвиг влево
>Сдвиг битов влево, умножение на 2^n.
>```
>0101 shl 1 = 1010
>```
>Зачем:
>1. Скорость — процессору сдвинуть быстрее чем умножение.
>2. Упаковка данных — несколько чисел в одно (RGB).

>[!question]- ushr — беззнаковый сдвиг вправо
>Слева вставляется 0 игнорируя знак.
>```kotlin
>val a = 8                       // 0000...00001000
>val b = a ushr 2                // 2 (0000...00000010)
>val c = (-1).ushr(1)            // огромное положительное Int
>```

>[!question]- где используется — RGB
>```kotlin
>val red = 255
>val green = 128
>val blue = 64
>val rgb = (red shl 16) or (green shl 8) or blue
>val green2 = (rgb shr 8) and 0xFF
>```

>[!question]- литералы
>```kotlin
>0b0101         // двоичный, префикс 0b
>0x0F           // шестнадцатеричный, префикс 0x
>// восьмеричных нет в Kotlin (в отличие от Java)
>```

>[!question]- разница && и and (на булеанах)
>`&&` — short-circuit, идиоматичный для логики.
>`and` для `Boolean` — тоже есть, но **не short-circuit**, всегда вычисляет оба операнда. Использовать только когда нужно вычислить оба (с побочными эффектами).
>Для битов на числах — только `and`/`or`/`xor`, `&` для чисел нет.
