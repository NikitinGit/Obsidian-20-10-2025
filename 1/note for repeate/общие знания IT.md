>[!question]- UFW 
>UFW расшифровывается как Uncomplicated Firewall﻿, что переводится на русский как «несложный фаервол»﻿ или «незамысловатый межсетевой экран»﻿. Это утилита для упрощённого управления настройками межсетевого экрана (брандмауэра) в Linux, которая служит интерфейсом для более сложного инструмента iptables.

>[!question]- Брандмауэр
>Брандмауэр﻿ — это система безопасности сети, которая контролирует и фильтрует входящий и исходящий сетевой трафик для защиты компьютера или сервера от несанкционированного доступа и потенциальных угроз. UFW облегчает настройку этих правил для управления, какие подключения разрешены или заблокированы.

>[!question]- iptables 
>UFW и iptables делают одно и то же по сути (управляют правилами файрвола), но работают на разном уровне абстракции.  Низкоуровневый инструмент, напрямую управляющий подсистемой netfilter﻿ в ядре Linux (цепочки INPUT/FORWARD/OUTPUT, таблицы filter/nat/mangle и т.д.).

>[!question]- telnet 
> утилита подключения к удаленному хосту 
> **может ли ваш сервер установить TCP-соединение с удалённым хостом на нужный порт**. 
> пример 
> ```
> telnet api.yookassa.ru 443
> ```
> Если соединение успешно — вы увидите: 
> ```
> Connected to api.yookassa.ru
> Escape character is '^]'.
> ```
> Если нет — будет:
> ```
> Connection timed out
> ```

>[!question]- NAT-firewall 
>сетевой фильтр - который стоит в инфраструктуре вашего VPS-провайдера и контролирует, куда могут уходить исходящие соединения с вашего сервера.
>Большинство VPS живут за NAT (как домашний Wi-Fi роутер): 
>Ваш VPS → NAT хостера → Интернет
>Хостер помещает ваши серверы в приватную сеть, а в Интернет они выходят через общий шлюз. 



Подходы в индустрии

  1. Google, Netflix, Amazon: Contract-Based + Fail-Fast

  Философия: "Не маскируйте проблемы защитой, пусть падает рано и громко"

  // НЕ добавляют nullsFirst(), потому что:
  // 1. Запрос гарантирует NOT NULL
  // 2. Если кто-то нарушит контракт - тест упадет сразу
  // 3. NPE - это правильный сигнал о нарушении инварианта

  // Вместо этого добавляют:
  @Contract("Contract: findFightReadyBattles MUST return battles with non-null eventDate")

  Преимущества:
  - Не скрывает баги
  - Меньше неявного поведения
  - NPE сразу показывает место проблемы

  2. Microsoft, Facebook: Defensive Programming

  Философия: "Production не должен падать, даже если контракт нарушен"

  // Добавляют nullsFirst() даже если "не нужно"
  .sorted(Comparator.comparing(
      BaseBattleDto::getBattleDate,
      Comparator.nullsFirst(Comparator.naturalOrder())
  ))

  Преимущества:
  - Система стабильнее
  - Защита от будущих изменений
  - Graceful degradation

  3. Современный подход (2024-2025): Tests + Static Analysis
