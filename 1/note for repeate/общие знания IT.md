
>[!question]-  JSON
>  текстовый формат для хранения структурированных данных и обмена ими. Чаще всего используется в вебе.
>  - Данные в JSON представлены в виде пар «ключ — значение».
>  - Для хранения данных в JSON используются две структуры — объекты и массивы. Объекты — неупорядоченные коллекция, массивы — упорядоченная.
>  - JSON5 расширяет возможности формата, но пока не поддерживается браузерами.
>  - JSON Schema используется для определения требований к данным: разработчик может указать обязательные поля и их допустимые форматы.
> - Для работы с JSON используется два основных подхода: сериализация, позволяющая преобразовать JavaScript-код или код другого языка программирования в JSON-формат, и десериализация, работающая наоборот: из JSON формата данные переводятся в обычный код.

>[!question]- UFW 
>UFW расшифровывается как Uncomplicated Firewall﻿, что переводится на русский как «несложный фаервол»﻿ или «незамысловатый межсетевой экран»﻿. Это утилита для упрощённого управления настройками межсетевого экрана (брандмауэра) в Linux, которая служит интерфейсом для более сложного инструмента iptables.

>[!question]- Брандмауэр
>Брандмауэр﻿ — это система безопасности сети, которая контролирует и фильтрует входящий и исходящий сетевой трафик для защиты компьютера или сервера от несанкционированного доступа и потенциальных угроз. UFW облегчает настройку этих правил для управления, какие подключения разрешены или заблокированы.

>[!question]- iptables 
>UFW и iptables делают одно и то же по сути (управляют правилами файрвола), но работают на разном уровне абстракции.  Низкоуровневый инструмент, напрямую управляющий подсистемой netfilter﻿ в ядре Linux (цепочки INPUT/FORWARD/OUTPUT, таблицы filter/nat/mangle и т.д.).

>[!question]- NAT-firewall 
>**Network Address Translation**  -  это технология, которая позволяет нескольким устройствам в локальной сети использовать один общий публичный IP-адрес для выхода в интернет.
>- **Как это работает:** Когда ваше устройство (телефон или ноутбук) отправляет запрос в интернет, NAT заменяет его частный IP-адрес (например, 192.168.1.5) на публичный адрес вашего роутера.
>сетевой фильтр - который стоит в инфраструктуре вашего VPS-провайдера и контролирует, куда могут уходить исходящие соединения с вашего сервера.
>Большинство VPS живут за NAT (как домашний Wi-Fi роутер): 
>Ваш VPS → NAT хостера → Интернет
>Хостер помещает ваши серверы в приватную сеть, а в Интернет они выходят через общий шлюз. 

# **Экранирование**
это способ замены управляющих или специальных символов их безопасными текстовыми эквивалентами. 
Основные правила в JSON (на 2026 год):

Для вставки специальных знаков внутри строковых значений используется обратный слеш (`\`): 

- **`\"`** — двойная кавычка (чтобы парсер не подумал, что строка закончилась).
- **`\\`** — обратный слеш (сам символ экранирования).
- **`\n`** — перенос строки (newline).
- **`\t`** — горизонтальная табуляция.
- **`\r`** — возврат каретки.
- **`\/`** — прямая косая черта (необязательно, но допустимо).
- **`\uXXXX`** — произвольный символ Unicode в 16-ричном коде (например, `\u00A9` для знака ©). 
Пример проблемы и решения:

- **Ошибка:** `"message": "Он сказал: "Привет""` — парсер увидит конец строки после двоеточия.
- **Правильно:** `"message": "Он сказал: \"Привет\""` — благодаря `\"` кавычки считаются частью текста.
# Серый ip адрес 
В 2026 году из-за дефицита IPv4-адресов большинство провайдеров по умолчанию выдают не просто динамический, а **«серый» (частный)** адрес. 

- В этом случае провайдер использует технологию **CGNAT** (Carrier-Grade NAT).
- Ваш роутер получает внутренний адрес сети провайдера, а в большой интернет вы выходите через один общий публичный IP-адрес вместе с сотнями других абонентов.

# CDN 
(Content Delivery Network) — это сеть серверов, распределённых по разным географическим точкам, которая ускоряет загрузку сайта.
  Вместо того, чтобы загружать все файлы (картинки, стили, скрипты) с одного сервера, CDN хранит копии этих файлов на множестве серверов по всему миру. Когда пользователь открывает сайт, файлы загружаются с ближайшего к нему сервера.

  В случае с вашим сайтом на Tilda:

  tildacdn.pub — это CDN-домен Tilda, где хранятся:
  - static.tildacdn.pub — статические файлы (CSS стили, JavaScript скрипты)
  - thb.tildacdn.pub — изображения и медиафайлы


# Подходы в индустрии

  1. Google, Netflix, Amazon: Contract-Based + Fail-Fast

  Философия: "Не маскируйте проблемы защитой, пусть падает рано и громко"

  // НЕ добавляют nullsFirst(), потому что:
  // 1. Запрос гарантирует NOT NULL
  // 2. Если кто-то нарушит контракт - тест упадет сразу
  // 3. NPE - это правильный сигнал о нарушении инварианта

  // Вместо этого добавляют:
  @Contract("Contract: findFightReadyBattles MUST return battles with non-null eventDate")

  Преимущества:
  - Не скрывает баги
  - Меньше неявного поведения
  - NPE сразу показывает место проблемы

  2. Microsoft, Facebook: Defensive Programming

  Философия: "Production не должен падать, даже если контракт нарушен"

  // Добавляют nullsFirst() даже если "не нужно"
  .sorted(Comparator.comparing(
      BaseBattleDto::getBattleDate,
      Comparator.nullsFirst(Comparator.naturalOrder())
  ))

  Преимущества:
  - Система стабильнее
  - Защита от будущих изменений
  - Graceful degradation

  3. Современный подход (2024-2025): Tests + Static Analysis
