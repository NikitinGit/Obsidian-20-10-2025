>[!question]- final внутри лямбда выражений
> переменная внутри метода может быть или final, или effectively final — то есть не меняться после присвоения:
>```
>int x = 5;           // effectively final
>Runnable r = () -> System.out.println(x); // ок
>```
  > На поля класса это не распространяется — их можно читать и менять:
  > ```
  > class Example {
  >      static int z;
  >    int counter = 0;
>
>      void run() {
>          Runnable r = () -> {
>             counter++;
>              z--;
>          }; // ок — поле класса, не локальная переменная
>          r.run();
>  }
  >}
  > ```
