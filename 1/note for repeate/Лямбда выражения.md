>[!question]- final внутри лямбда выражений
> переменная внутри метода может быть или final, или effectively final — то есть не меняться после присвоения:
>```
>int x = 5;           // effectively final
>Runnable r = () -> System.out.println(x); // ок
>```
  > На поля класса это не распространяется — их можно читать и менять:
  > ```
  > class Example {
  >      static int z;
  >    int counter = 0;
>
>      void run() {
>          Runnable r = () -> {
>             counter++;
>              z--;
>          }; // ок — поле класса, не локальная переменная
>          r.run();
>  }
  >}
  > ```

>[!question]- стандартные функциональные интерфейсы java.util.function
>| Интерфейс | Сигнатура | Что делает |
>|---|---|---|
>| `Predicate<T>` | `boolean test(T t)` | проверка условия |
>| `Function<T,R>` | `R apply(T t)` | преобразование T → R |
>| `BiFunction<T,U,R>` | `R apply(T t, U u)` | преобразование (T,U) → R |
>| `Consumer<T>` | `void accept(T t)` | действие над объектом без возврата |
>| `Supplier<T>` | `T get()` | поставщик объекта без параметров |
>| `UnaryOperator<T>` | `T apply(T t)` | Function<T,T> — один тип |
>| `BinaryOperator<T>` | `T apply(T t1, T t2)` | BiFunction<T,T,T> — один тип |
>
>```java
>Predicate<Integer> testInt = x -> x % 3 == 1;
>testInt.test(4); // true
>
>BinaryOperator<Integer> multiply = (x, y) -> x * y;
>multiply.apply(3, 5); // 15
>
>UnaryOperator<Integer> square = x -> x * x;
>square.apply(5); // 25
>
>Function<Function1, Function2> f1ToF2 = Function1::getF2; // ссылка на нестатический метод
>f1ToF2.apply(f1); // f1.getF2()
>
>BiFunction<Integer, Function1, Function2> intWithF1ToF2 = (x, y) ->
>        new Function2(y.getName() + ": " + (x * 2));
>
>Consumer<Function2> consumerF2 = x -> x.setName("consumerF2");
>consumerF2.accept(f2); // меняет f2, ничего не возвращает
>```
>
>**Ссылка на нестатический метод через класс** — объект передаётся первым аргументом:
>```java
>Function<Function1, Function2> f1ToF2 = Function1::getF2;
>// компилятор разворачивает в: f1 -> f1.getF2()
>```
