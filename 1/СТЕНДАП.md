SMS, code=520 «Шаблон для рассылки не найден» — ошибка шлюза smsint.ru. Российские SMS-операторы требуют заранее зарегистрированный и промодерированный шаблон текста. Наш текст (challenge.notification.sms.message = «По вашему вызову на бой есть обновление…») у провайдера как шаблон не заведён → 520. Это задача на стороне провайдера (зарегистрировать шаблон рассылки), а не код.  Email, SSLHandshakeException ... smtp.gmail.com:465 — локальная проблема подключения/TLS к Gmail. Типичные причины: порт 465 заблокирован сетью/ISP/файрволом (на локалке частое); рассогласование порт↔SSL: 465 = implicit SSL (mail.smtp.ssl.enable=true), 587 = STARTTLS (mail.smtp.starttls.enable=true). Если конфиг не соответствует порту — рукопожатие рвётся; антивирус/прокси, перехватывающий TLS. Тоже инфраструктура/конфиг, не код.

>[!question]- Как  отдельно запустить up() / down()
> Запустить отедельно  миграцию  (up() )
>```
>  php artisan migrate --path=/database/migrations/2025_12_24_000000_migrate_judges_scores_to_new_structure.php
>```
>Запуск метода down() (откатить миграцию):
> Вариант 1: Откатить последнюю пакетную миграцию
>```
>  php artisan migrate:rollback
>```
>Откатить 2 миграции
>```
>Bash(cd /home/igor/IdeaProjects/strikerstat/backend-php && php artisan migrate:rollback --step=3 2>&1 | tail -15)
>```
>
> Вариант 2: Откатить только эту конкретную миграцию
>```
>  php artisan migrate:rollback --path=/database/migrations/2025_12_24_000000_migrate_judges_scores_to_new_structure.php
>```
> Вариант 3: Откатить последнюю одну миграцию 
> ```
>  cd /home/igor/IdeaProjects/strikerstat/backend-php && php artisan migrate:rollback --step=1 2>&1 | tail -15
>   cd /home/igor/IdeaProjects/strikerstat/backend-php && php artisan migrate:rollback --step=1 2>&1 | tail -15
> ```
>  Посмотреть статус миграций
>  ```
>   php artisan migrate:status
>  ```
>   Откатить и заново применить миграцию (down + up)
>   ```
>    php artisan migrate:refresh --path=/database/migrations/2025_12_24_000000_migrate_judges_scores_to_new_structure.php
>   ```
